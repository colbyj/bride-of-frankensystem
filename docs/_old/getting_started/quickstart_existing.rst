Quickstart: Run an Existing Experiment
=======================================

This guide walks through downloading the BOFS examples repository and running the *minimal example* — a small but complete BOFS project. It's the fastest way to see a working experiment end-to-end before deciding what your own project should look like.

If you'd rather start from a blank project generated by the ``BOFS init`` wizard, see :doc:`quickstart_create` instead. For descriptions of the other example projects in the repo (advanced, p5, unity), see :doc:`/examples/example_projects`.

Prerequisites
-------------

Make sure BOFS is installed before continuing — see :doc:`installation`.

Download the Examples
---------------------

Download the examples repository as a ZIP from `the BOFS examples GitHub page <https://github.com/colbyj/bride-of-frankensystem-examples/archive/refs/heads/master.zip>`_ and unzip it somewhere convenient. You can also browse the source `on GitHub <https://github.com/colbyj/bride-of-frankensystem-examples>`_.

Inside the unzipped folder you'll find several example projects. We'll use ``minimal_example/``.

The Files
---------

The minimal example consists of three things you'll edit when building your own project plus a folder of questionnaires:

============================================  =============================================================================
Path                                          Description
============================================  =============================================================================
``minimal_example/minimal.toml``              Configuration: database, page list, admin password, port, and so on.
``minimal_example/consent.html``              The HTML shown on the consent page.
``minimal_example/questionnaires/*.json``     One JSON file per questionnaire, defining the questions and their layout.
============================================  =============================================================================

Open ``minimal.toml`` to see how the project is wired together. Two settings matter most for now:

* ``ADMIN_PASSWORD`` is the password for the ``/admin`` panel (``example`` for this project).
* ``PAGE_LIST`` is the ordered list of pages the participant moves through. For the minimal example it's: consent → an example questionnaire (shown twice with different "tags") → a questionnaire that performs calculations → an end page with a completion code.

For a complete description of every setting, see :doc:`/reference/config_options`.

Run the Example
---------------

Open a terminal in the ``minimal_example/`` folder and run:

.. code-block:: bash

    BOFS run minimal.toml -d

The ``-d`` flag enables debug mode, which adds navigation controls at the bottom of every page and surfaces more detailed error messages. You should see output like:

.. code-block:: text

    Loading blueprint: BOFS.admin
    Loading blueprint: BOFS.default
    BOFS.default: `models.py` loaded!
    Listening on http://0.0.0.0:5003
    Preview locally at http://127.0.0.1:5003

Open http://127.0.0.1:5003 in your browser to view the experiment as a participant would see it. Press ``Ctrl+C`` in the terminal to stop the server.

The Admin Panel
---------------

Every BOFS project includes an admin panel at ``/admin``. For this project it's at http://127.0.0.1:5003/admin — log in with the password ``example`` (defined in ``minimal.toml``).

The admin panel lets you watch participant progress in real time, export collected data as CSV, preview questionnaires, and browse the underlying database tables. Full details: :doc:`/getting_started/admin`.

.. tip::
    When testing a project locally, use a private/incognito browser window or visit ``/restart`` to clear your session and replay the experiment from the beginning.
